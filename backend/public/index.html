Adaptive Gateway Routing for Card Payments

A live demo of merchant-side payment orchestration: transaction input, fraud-risk scoring, approve-or-block logic, and gateway selection.

Express API Flask ML Service Random Forest

Flow

  1. Merchant checkout captures payment details
  2. Fraud score is generated using the ML service
  3. Low-risk payments are routed to the best-fit gateway
  4. High-risk payments are blocked before routing

Architecture

Merchant-Layer Orchestration

The checkout UI talks to Express. Express calls Flask for fraud scoring, then routes approved payments to the best gateway.

Algorithm

Random Forest + Threshold

The model returns a fraud probability. If it exceeds the threshold, the transaction is blocked. Otherwise, routing logic picks a gateway.
